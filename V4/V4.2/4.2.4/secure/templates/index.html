    📖 What is CRLF Injection?
  
      
  
      
    
      
      
      📡
      
      
      HTTP Headers & CRLF
      
      
      
        In HTTP/1.1, headers are separated by CRLF (\r\n) sequences.
        Each header line is: Name: Value\r\n.
        If an attacker injects \r\n into a header value, they effectively
        terminate the current header and begin a new one — Header Injection.
      
      
    
      
    
      
      
      ✂️
      
      
      Response Splitting
      
      
      
        By injecting \r\n\r\n (double CRLF), an attacker can split an HTTP
        response into two. Proxies and caches may cache the second injected response,
        serving malicious content to other users — Cache Poisoning.
      
      
    
      
    
      
      
      📝
      
      
      Log Injection
      
      
      
        Server logs often record header values. Injecting \n into a
        header value allows an attacker to write fake log entries, hiding their
        tracks or framing other users by forging log lines.
      
      
    
      
    
      
      
      🔏
      
      
      HTTP/2 & HTTP/3 Protection
      
      
      
        RFC 9113 §8.2.1 (HTTP/2) and RFC 9114 §4.2 (HTTP/3) explicitly forbid
        CR and LF in header fields. Binary framing means these characters have no
        special meaning at the transport layer. But ASVS 4.2.4 requires
        application-level validation too.
      
      
    
      
  
      

      



      
  
      
    🛡️ How the Secure App Validates Headers
  
      
  
      
    
      
      
      Validation Logic (Python)
      
      
      
        A before_request hook runs on every incoming request.
        It inspects every header name and value using a regex that matches
        any CR (0x0d) or LF (0x0a) byte.
        If any violation is found, the request is immediately rejected with HTTP 400.

      app.py — Secure Validation
      
      
      CRLF_PATTERN = re.compile(r"[\r\n]")

def validate_headers(headers: dict) -> list[str]:
    violations = []
    for name, value in headers.items():
        if CRLF_PATTERN.search(name):
            violations.append(f"Header name contains CR/LF")
        if CRLF_PATTERN.search(value):
            violations.append(f"Header '{name}' value contains CR/LF")
    return violations

@app.before_request
def enforce_no_crlf_in_headers():
    violations = validate_headers(dict(request.headers))
    if violations:
        return jsonify({"status": "REJECTED", "violations": violations}), 400
