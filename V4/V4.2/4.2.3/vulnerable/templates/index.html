Control Information

ASVS Control: 4.2.3
Level: 3 (Comprehensive)
Category: HTTP Message Structure Validation
Implementation: SECURE (PASS)

Requirement

Verify that the application does not send nor accept HTTP/2 or HTTP/3 messages with connection-specific header fields such as Transfer-Encoding to prevent response splitting and header injection attacks.

Why This Matters

HTTP/2 Architecture: HTTP/2 fundamentally changed HTTP. Instead of one TCP connection per request/response, HTTP/2 multiplexes multiple logical streams over a single connection. This makes connection-specific headers meaningless and dangerous.

Forbidden Headers (RFC 7540):

  • Transfer-Encoding - HTTP/2 uses frame-level length, not chunking
  • Connection - Connection state managed by GOAWAY frames
  • Keep-Alive - HTTP/2 connections are persistent by design
  • Upgrade - Protocol upgrade negotiated differently
  • Proxy-Connection - Non-standard; obsolete in HTTP/2
  • Trailer - Trailers handled differently in HTTP/2

Attack Vectors:

  • Header Injection: Inject CRLF sequences to add arbitrary headers
  • Response Splitting: Split response and inject attacker content
  • Request Smuggling: Confuse different protocol layers about message boundaries
  • Cache Poisoning: Inject headers used in cache keys
  • Protocol Confusion: Downgrade from HTTP/2 or cause parsing errors

Relevant Standards

  • RFC 7540 Section 8.1.2 - HTTP/2 Specification
  • RFC 3986 - URI Generic Syntax
  • CWE-444 - Inconsistent Interpretation of HTTP Requests
  • CWE-113 - Improper Neutralization of CRLF Sequences

curl Commands

            
Test with curl (requires HTTP/2 support):

            

# Test with Transfer-Encoding header (should be rejected)
curl -H "Transfer-Encoding: chunked" \
  https://localhost:8000/api/test

# Test with Connection header (should be rejected)
curl -H "Connection: close" \
  https://localhost:8000/api/test

# Test with multiple forbidden headers
curl -H "Transfer-Encoding: chunked" \
  -H "Connection: close" \
  -H "Keep-Alive: timeout=5" \
  https://localhost:8000/api/test

# Valid request (should succeed)
curl -H "Accept: application/json" \
  https://localhost:8000/api/test
